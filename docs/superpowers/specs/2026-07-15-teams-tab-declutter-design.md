# Разгрузка вкладки «Команды» → «Мои команды» в профиле

**Дата:** 2026-07-15
**Репозиторий:** `DiscordForFemka-Frontend` (изменения только на фронте)
**Ветка:** `feat/teams-tab-declutter`

## Проблема

Пользователи жалуются, что вкладка **«Команды»** перегружена. Сейчас это
публичный каталог *всех* команд (`/teams` → `TeamsListPage`): searchable,
пагинированная сетка карточек каждой команды. Визуально тяжёлый экран, который
для рядового пользователя избыточен.

При этом сами команды нужны как **стаки** — их регистрируют на турниры.

## Ключевые факты (из анализа кода)

- **Регистрация на турнир не использует публичный каталог.** Кнопка
  «Зарегистрироваться» в `TournamentDetailsPage` берёт команды из `me.teams`
  и регистрирует ту, где пользователь `CAPTAIN` и `teamStatus === 'ACTIVE'`.
  То есть каталог всех команд можно убрать, не ломая регистрацию.
- **Приглашения уже полностью реализованы.** Есть `/me/invites`
  (`MyInvitesPage`) с accept/decline, nav-ссылка «Приглашения» с бейджем
  количества (появляется при наличии pending) и пункт в дропдауне аватара.
  Добавлять ничего не нужно.
- **Пробел:** `ProfilePage` сейчас **не показывает** команды пользователя.
  Единственная точка входа к своему стаку сегодня — найти его в публичном
  каталоге. Поэтому при удалении каталога нужна новая точка входа.
- `me.teams` — это `TeamMembershipDto[]` с полями `teamId`, `name`, `tag`,
  `role` (`TeamMemberRole`), `teamStatus` (`TeamStatus`). Данные уже приходят
  во `me`, **дополнительных запросов не требуется**.

## Решение

Frontend-only. Бэкенд (team/invite API) не трогаем.

### 1. Убрать публичный каталог команд

- `src/components/Header.tsx` — удалить nav-ссылку **«Команды»** (`/teams`).
- `src/App.tsx` — удалить роут `/teams` и импорт `TeamsListPage`.
  **Оставить** `/teams/new` (`TeamCreatePage`) и `/teams/:id`
  (`TeamDetailsPage`).
- Удалить `src/pages/TeamsListPage.tsx`.
- Удалить ставший мёртвым после этого код: хук `useTeamsList` в
  `src/lib/queries.ts`, а также `getTeamsPage` и тип `TeamsPageParams`
  (единственный потребитель — удаляемая страница). Перед удалением
  подтвердить grep'ом, что других потребителей нет.

### 2. Блок «Мои команды» в профиле

В `src/pages/ProfilePage.tsx` добавить секцию (Card) со списком стаков из
`me.teams`:

- Для каждой команды:
  - название + тег → `TeamNameLink` на `/teams/:teamId`;
  - бейдж роли — `TEAM_MEMBER_ROLE_LABEL[role]` (Капитан / Игрок);
  - бейдж статуса — `TEAM_STATUS_LABEL[teamStatus]`;
  - кнопка «Открыть» → навигация на `/teams/:teamId`.
- В шапке секции — кнопка **«+ Создать команду»** → `/teams/new`.
  `disabled`, если профиль `INACTIVE` (как в существующей логике
  `TeamsListPage`/`Header`), с тем же `title`.
- Пустое состояние: «У вас пока нет команды» + та же кнопка создания.
- Стиль — существующая дизайн-система Editorial Clean (токены ink/line/brand,
  компоненты `Card`/`Badge`/`Button`), как на остальном профиле.

### 3. Что остаётся нетронутым

- Регистрация на турнир (`TournamentDetailsPage`, читает `me.teams`).
- `TeamDetailsPage` (`/teams/:id`), `TeamNameLink`, `TeamHoverCard`, ссылки на
  команды из ростеров турниров/матчей.
- Приглашения (`/me/invites`, бейдж в шапке и дропдауне).
- Админский раздел команд (`/admin/teams`, `useAdminTeams`) — не связан с
  публичным каталогом.

## Проверка

- `npm run build` / typecheck — убедиться, что нет мёртвых импортов и
  неиспользуемых символов после удаления каталога.
- Ручной прогон в браузере:
  - в шапке нет пункта «Команды», прямой переход на `/teams` больше не
    открывает каталог;
  - профиль показывает блок «Мои команды» с корректными ролями/статусами и
    рабочими ссылками; пустое состояние отображается у пользователя без команд;
  - кнопка «Создать команду» ведёт на `/teams/new`;
  - страница команды `/teams/:id` открывается по ссылке;
  - регистрация капитана на турнир по-прежнему работает.

## Вне рамок (YAGNI)

- Никаких изменений в бэкенде.
- Не добавляем новый «мой» список команд отдельной вкладкой — точка входа
  живёт в профиле.
- Не трогаем сам `TeamDetailsPage` и флоу приглашений.
